Migration Guide

v1.0.0 of Open MCT introduces some major changes under the hood that may affect any applications built with Open MCT. The build environment is now based on Webpack, and a lot of our Angular 1.x code has been replaced with vanilla JavaScript and Vue.js.

To the extent possible we have tried to maintain backwards compatibility. The guide below details the breaking changes that we are aware of, so if you come across any compatibility problems not captured here please file issues in our GitHub repository. These breaking changes are minor, and limited to the scaffolding in the index.html file that includes Open MCT.


Stylesheets and "Asset Path"

You will need to specify the location that the compiled openmct assets (eg. the javascript, stylesheets, etc.) are hosted on your web server. This can be specified using the setAssetPath() function. This was also the case with previous builds of Open MCT, but the location of these assets in the Open MCT node dependency has changed from previous releases. This path is used by various plugins to locate the stylesheets etc. appropriate for the installed theme. As such, issues with this path may manifest as the application rendering the markup with no styling.

Assuming that your node_modules directory is hosted at the root level of your webserver, the path to the Open MCT assets directory will be node_modules/openmct/dist/. If it is not hosted at the root of your server, you will need to adjust this path accordingly.

Also, if you have a route configured on your server that aliases node_modules/openmct/dist (eg as just openmct) then you will need to adjust the paths above accordingly.

Eg. openmct.setAssetPath(‘node_modules/openmct/dist/’);


startup-base.css

The stylesheet startup-base.css is no longer required, and any link tag referencing it can be removed.


Favicons

The Open MCT favicons have moved from dist/platform/commonUI/general/res/images/favicons/ to dist/favicons/ If you are linking directly to the favicons from the node_modules directory, then the full paths will change from

node_modules/openmct/dist/platform/commonUI/general/res/images/favicons/favicon-32x32.png
node_modules/openmct/dist/platform/commonUI/general/res/images/favicons/favicon-96x96.png
node_modules/openmct/dist/platform/commonUI/general/res/images/favicons/favicon-16x16.png

To

node_modules/openmct/dist/favicons/favicon-32x32.png
node_modules/openmct/dist/favicons/favicon-96x96.png
node_modules/openmct/dist/favicons/favicon-16x16.png

Again, if you have a route configured on your server that aliases node_modules/openmct/dist (eg as just openmct) then you will need to adjust the paths above accordingly.


Splash Screen

The splash screen has been removed for now. The associated markup should be removed -

<div class="l-splash-holder s-splash-holder">
    <div class="l-splash s-splash"></div>
</div>
